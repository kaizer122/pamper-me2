# API runs Hono on Bun; Zod is the single validation language

`apps/api` is a Bun service built on Hono with `@hono/zod-validator` for request validation. `packages/contracts` re-exports the inferred client type from `hc<typeof app>`, giving all web apps end-to-end type safety against the API. We considered Elysia (faster on Bun, Eden Treaty has the cleanest end-to-end typing in TypeScript) but rejected it because Elysia's first-class validation language is TypeBox, and we have already standardized on Zod for shared client+server schemas in `packages/validation`. Forcing a second validation library or a Zod-adapter that's a second-class citizen would compromise the single-source-of-truth stance more than Hono's slightly less ergonomic RPC story costs us. Hono's multi-runtime support is a free hedge we don't plan to use but don't pay for. Fastify and NestJS were rejected as poor fits for Bun-first development.
