# Frontend stack: TanStack Start for marketplace-web, TanStack Router (Vite) for business-web and admin-web

All three web apps run on the TanStack ecosystem. `marketplace-web` uses TanStack Start (SSR + file-based type-safe routing) for SEO-critical Client-discovery pages. `business-web` and `admin-web` use TanStack Router on Vite as SPAs. TanStack Query is the canonical data-fetching layer in all three, paired with the auto-generated Hono `hc` client from `packages/contracts` for end-to-end type safety. We considered Next.js everywhere (rejected: three Next.js apps where two don't need its SSR is paying tax for nothing) and a Next.js + Vite split (rejected: maintaining two frontend toolchains has a real cognitive cost we don't need to absorb). The deliberate risk we accept: TanStack Start is materially newer than Next.js, with a thinner ecosystem of recipes and deployment guides. We accept this risk because the alternative is fragmenting our type-flow story between Next.js conventions and TanStack conventions, and because Vite's dev velocity compounds across three apps in active development. A future engineer reading the code may assume Next.js was the obvious choice — it was deliberately rejected for ecosystem unity.
