# ServiceOffering carries a single price and duration in v1; per-Practitioner overrides deferred

Every ServiceOffering has one price and one duration that apply to all eligible Practitioners. Businesses that want senior/junior pricing model it as two separate Offerings ("Haircut — Senior", "Haircut — Junior") with different eligibility lists. We considered per-Practitioner price/duration overrides — supported by Treatwell, Fresha, and Vagaro — but deferred them: the override-grid UI is one of the gnarliest screens in salon SaaS, and the "two Offerings" workaround is a 5-minute setup cost. The schema is forward-compatible: a `PractitionerOfferingOverride` table can be added later without migrating existing data. This is a deliberate deviation from competitor norms; a future engineer reading the code may assume we forgot — we didn't.
