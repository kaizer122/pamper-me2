# 15-minute slot grid platform-wide; per-Offering `bufferAfter` only in v1

Client-facing slot start times are aligned to a fixed 15-minute grid platform-wide, not configurable per Business. The Practitioner's actual calendar can hold any minute; only the Client browse view is gridded. Each ServiceOffering carries an optional `bufferAfter` (extra calendar time consumed after the Booking ends, invisible to the Client). The Practitioner's calendar block for a Booking is `duration + bufferAfter`. We deferred `bufferBefore` (rare in practice; typically absorbed by the previous Booking's `bufferAfter`) and multi-stage processing-time Offerings (active/passive/active — needed for chemical color services where the Practitioner is free during processing). Multi-stage is a real loss for v1: salons doing complex color jobs will have to inflate `bufferAfter` and lose efficiency. We accept this for v1 and treat multi-stage as a deliberate v1.5 workstream once we have customer feedback to inform the design.
