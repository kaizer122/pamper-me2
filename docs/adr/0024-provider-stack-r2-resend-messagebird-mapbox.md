# Provider stack: R2 (storage), Resend (email), MessageBird (SMS, swappable), Mapbox (maps)

External providers selected for v1: Cloudflare R2 for object storage (S3-compatible, no egress fees — matters for Marketplace image serving), Resend for transactional + marketing email (TypeScript-native SDK, React Email integration), MessageBird (or Vonage; both viable) for SMS at v1, Mapbox for maps and geocoding (better Tunisia street-level coverage than Google in our target neighborhoods, cheaper at expected volume, good React components). The deliberate non-obvious decision is SMS: Twilio was rejected on per-message cost; local Tunisian aggregators (Tunisie Telecom Bulk, Orange Tunisie, Smsfacile.tn, etc.) are materially cheaper but have crustier integrations (no TS SDK, HTTP/SOAP). v1 ships on MessageBird because DX velocity matters more than per-message cost at our launch volume; v1.5 plans a swap to a local aggregator once SMS volume justifies the operational cost. To make that swap a one-env-var change, the application-side abstraction is a `SmsProvider` interface (likely in a `packages/notifications` package once SMS-related code grows beyond a single file). All other providers are forward-replaceable through similar narrow interfaces but no swap is planned.
